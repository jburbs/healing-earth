# Healing Earth with Technology — Site Architecture & Update Guide

## Overview

This is a static HTML archive of 182 Substack newsletter issues, deployed on Netlify at **healingearthwithtech.com**. The GitHub repo is **jburbs/healing-earth**. There is no build step — Netlify serves the files directly from the repo root.

The live Substack at **healingearth.substack.com** retains the subscriber list and can be rebranded (e.g., to Modulus Water) independently. This archive site serves as the permanent reference library for the original Healing Earth content.

## Repository Structure

```
/
├── index.html              ← Landing page (hero, topic cards, reading paths, archive)
├── about.html              ← Expanded About page
├── style.css               ← All styles (landing page + post pages share this file)
├── app.js                  ← Landing page logic (tabs, filtering, search, popularity sort)
├── categories.js           ← Post → category mapping + series definitions
├── nav-data.js             ← Precomputed prev/next navigation per category
├── post-nav.js             ← Post page navigation (arrows, category badges)
├── search.js               ← Search index and matching logic
├── search-universal.js     ← Universal search (nav bar search on post pages)
├── search-index.json       ← Prebuilt search index
├── popularity-data.js      ← Substack engagement rankings (1–50, rest = 999)
├── favicon.png
├── images/
│   └── wordmark.png        ← Horizontal logo used in nav
├── posts/
│   └── YYYY-MM-DD_NNN_Title/
│       ├── index.html      ← The post
│       └── images/         ← Local copies of all post images
├── _organized_archive/     ← Metadata JSONs (not served, reference only)
└── post_manifest.json      ← Original export manifest
```

## Key Files and What They Do

### index.html
The landing page. Structure:
1. **Sticky nav** — wordmark + About link (no Subscribe)
2. **Hero section** — title, tagline, author line, stats
3. **Topic cards** — 6 cards (climate, solutions, policy, ai, science, notes); clicking one filters the archive and scrolls down
4. **Reading paths** — 3 curated entry points (Core Argument, Engineering Deep-Dives, Science & Society)
5. **Full Archive** — tab bar (All / Series / Climate / Solutions / Policy / AI / Science), search box, and all 182 post `<li>` elements
6. **Footer** — dark green with epigraph

The archive list is the canonical source of all posts. Each `<li>` has a `data-folder` attribute matching its folder name in `posts/`.

### style.css
Shared by both the landing page and all 182 post pages. Design tokens are in `:root` at the top. The post pages load this same file (via `../../style.css?v=6`), so any changes to shared styles (fonts, colors, nav, footer) affect everything.

**Version parameter**: Bump `?v=N` in both `index.html` and post pages when changing CSS to bust Netlify's cache.

### app.js
Handles:
- **Popularity sorting** — reads `postPopularity` from `popularity-data.js`, stamps each `<li>` with `data-popularity`, sorts on load and on tab switch
- **Tab filtering** — shows/hides posts by category using `postCategories` from `categories.js`
- **Series view** — renders series cards from `seriesData`
- **Search** — delegates to `searchPosts()` from `search.js`
- **Topic card clicks** — intercepts clicks, calls `filterByCategory()`, scrolls to `#archive`

### categories.js
Two exports:
- `postCategories` — maps folder name → array of category strings (e.g., `["climate", "solutions"]`)
- `seriesData` — defines 18 multi-part series with ordered part lists

### nav-data.js
Precomputed arrays for each category listing posts in chronological order. Used by `post-nav.js` to render prev/next arrows on post pages.

### popularity-data.js
Maps folder names → Substack popularity rank (1 = most popular). Top 50 posts are ranked; the rest default to 999. Generated by scraping `healingearth.substack.com/archive?sort=top`.

### post-nav.js
Runs on every post page. Reads `nav-data.js` and `categories.js` to render:
- Top strip: `← prev title | Category badge | next title →`
- Bottom strip: same arrows + "Filed under:" badges

Uses case-insensitive folder matching (required because Netlify lowercases URL paths but the data files use mixed case).

## How to Update

### Adding a new post

1. **Create the folder**: `posts/YYYY-MM-DD_NNN_Title/` with `index.html` and `images/`
2. **Add to `index.html`**: Insert a new `<li data-folder="YYYY-MM-DD_NNN_Title">` in the archive `<ul>` (currently the `<li>` elements are in chronological order in the HTML source, though they display sorted by popularity)
3. **Add to `categories.js`**: Map the folder name to its categories in `postCategories`
4. **Update `nav-data.js`**: Add the post to the appropriate category arrays (maintains chronological order for arrow navigation)
5. **Update `search-index.json`**: Add an entry with the post's title, excerpt, and content for search
6. **Optionally update `popularity-data.js`**: Add the folder with rank 999 (or a real rank if known)
7. Commit and push to `main`

### Editing the landing page content

- **Hero text**: Edit the `<section class="hero">` block in `index.html`
- **Topic card descriptions**: Edit the `.topic-card` elements
- **Reading paths**: Edit the `.path-card` elements; update the `<a href>` to point to the correct post folder
- **Stats line**: Update the hero-stats `<p>` (issue count, date range, hiatus status)

### Changing the design

All design tokens are in `style.css` under `:root`:
```css
--green-deep: #1a3a2a;
--green-accent: #2d6a4f;
--bg-warm: #faf8f5;
--font-serif: 'Libre Baskerville', Georgia, serif;
--font-sans: 'Source Sans 3', -apple-system, sans-serif;
```

Google Fonts are loaded in `index.html` and `about.html` `<head>`. Post pages inherit fonts from `style.css` but don't load Google Fonts (they fall back to Georgia/system sans).

### Updating popularity rankings

1. Go to `healingearth.substack.com/archive?sort=top`
2. Scroll to load all posts
3. Extract titles in order (browser console or scraper)
4. Match titles to local folder names
5. Update `popularity-data.js`

## Deployment

**Netlify** deploys automatically from the `main` branch of `jburbs/healing-earth`. There is no build command — Netlify serves files as-is.

**Preview branches**: Push any branch and create a PR; Netlify generates a deploy preview at `https://<branch>--healingearthwithtech.netlify.app/`.

**DNS**: `healingearthwithtech.com` points to Netlify.

**Important Netlify behavior**: Netlify lowercases all URL paths. This is why `post-nav.js` and `app.js` use case-insensitive matching for folder names. If you add a post with uppercase letters in the folder name, the navigation will still work, but the canonical URL on Netlify will be lowercase.

## Rebranding the Live Substack

The subscriber list lives on `healingearth.substack.com`. To rebrand for Modulus Water:

1. **Substack settings** → change publication name, logo, description, and About page
2. **Custom domain** (optional) → Substack supports custom domains if you want e.g. `newsletter.moduluswater.com`
3. **First rebranded post** — write a transitional post explaining the pivot: the Healing Earth archive lives at `healingearthwithtech.com`, and new content will focus on Modulus Water / water technology
4. **Cross-link** — the archive site's About page and footer can link to the rebranded Substack; the Substack can link back to the archive for readers who want the climate/energy back-catalog
5. **Subscriber expectations** — existing subscribers signed up for climate/energy content, so the transition post should set expectations and offer an easy unsubscribe for those not interested in the new direction

The two properties (static archive site + live Substack) are completely independent. Neither depends on the other to function.

## File Sizes and Performance

The site is ~150 MB total (mostly post images). Individual pages are lightweight HTML with no framework. The landing page loads ~7 scripts (all deferred), which is acceptable for a static site. There is no JavaScript framework, bundler, or build tool — everything is vanilla JS and hand-authored HTML/CSS.
